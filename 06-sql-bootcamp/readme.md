# 💾 SQL Bootcamp
> Практика работы с PostgreSQL: от базовых запросов до оконных функций, CTE, материализованных представлений и DML-операций. Все решения протестированы на учебной реляционной модели.

## 🎯 Цель
Освоить написание сложных аналитических запросов, оптимизацию через CTE и оконные функции, управление состоянием БД и работу с множествами.

## 🛠 Ключевые темы
| День | Навыки |
|------|--------|
| **Day 01** | `UNION/EXCEPT/INTERSECT`, подзапросы в `SELECT/FROM`, `CROSS JOIN`, `IN vs EXISTS`, фильтрация по диапазонам |
| **Day 02** | `LEFT/RIGHT/FULL JOIN`, рекурсивные `CTE`, `generate_series()`, поиск пропущенных дат, обработка `NULL` |
| **Day 03** | `INSERT/UPDATE/DELETE`, безопасная модификация через `JOIN`, массовая вставка `INSERT-SELECT`, динамический расчёт `PK` |
| **Day 04** | Представления (`VIEW`), материализованные представления (`MATERIALIZED VIEW`), симметричная разность, расчёт скидок, `REFRESH` |
| **Day 05** | Оконные функции: `ROW_NUMBER`, `RANK`, `DENSE_RANK`, `LAG/LEAD`, `NTILE`, фреймы (`ROWS/RANGE`), агрегация с `OVER()` |
| **Day 06** | `DDL/DML`, ограничения (`CHECK`, `UNIQUE`, `FOREIGN KEY`), последовательности (`SEQUENCE`), комментарии к схеме (Data Governance) |
| **Day 07** | Индексы (`B-Tree`, многоколоночные, частичные, функциональные), `EXPLAIN ANALYZE`, анализ планов запросов, оптимизация производительности |
| **Day 08** | Рекурсивные `CTE`, моделирование графов, задача коммивояжёра (TSP), поиск оптимальных маршрутов средствами SQL |

## ✅ Что реализовано
- Написание запросов без `IN/EXISTS` (через `JOIN` и операторы множеств)
- Оптимизация: замена вложенных подзапросов на `CTE`, генерация календарей через `generate_series`
- DML-операции: обновление/удаление связанных данных, массовая вставка с автоинкрементом
- Аналитика: ранжирование, сдвиги, скользящие суммы, процентили через оконные функции
- Управление схемой: создание `VIEW`/`MATERIALIZED VIEW`, инкрементальное обновление, очистка объектов
- Целостность данных: настройка `CHECK`/`UNIQUE`/`NOT NULL`, управление последовательностями для PK
- Оптимизация запросов: анализ `EXPLAIN ANALYZE`, замена `Seq Scan` на `Index Only Scan`, покрытие запросов индексами
- Алгоритмический SQL: рекурсивные обходы графов, решение задачи коммивояжёра (TSP) без внешних библиотек
